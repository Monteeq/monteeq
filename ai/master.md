# Monteeq AI Development Rules

## Core Rules
- Never redesign the UI
- Preserve current glassmorphism aesthetic
- Never rewrite CSS modules
- Never change website descriptions unnecessarily
- Keep mobile-first responsiveness
- Preserve current layouts

## Performance Rules
- Use lazy loading
- Use React.memo for heavy cards
- Optimize API payloads
- Avoid unnecessary rerenders
- Use React Query caching

## Backend Rules
- Use async FastAPI
- Optimize Supabase queries
- Never use SELECT *
- Add indexes for large tables

## Frontend Stack
- React + Vite
- CSS Modules
- React Query

## Backend Stack
- FastAPI
- Supabase
- Redis
- AWS S3

## UX Goals
- Cinematic
- Smooth
- Fast on low-end Android
- Premium streaming platform feel


## Commit
- always commit code after changes
- use meaningful commit messages
- keep commit messages short and concise
- use /speech.md for better commit messages
